On 3/29/22 React released version 18. One of the most significant changes made is the way the app is rendered. Instead of using ReactDOM.render(), a new method called createRoot should be used. React v18 will ship with both the new API and legacy API, so if you continue using the following syntax when creating a react app (create-react-app will also use this by default): 


import React from 'react';
import ReactDOM from 'react-dom';
import App from './App';

ReactDOM.render(<App />, document.getElementById('root'));

The above will work fine for now, but React will run v17 instead of 18, and you will see this warning below: 

   Warning: ReactDOM.render is no longer supported in React 18. Use createRoot instead. 
    Until you switch to the new API, your app will behave as if it's running React 17. 
    Learn more: https://reactjs.org/link/switch-to-createroot

To get rid of this warning, use createRoot instead:


import React from 'react';
import ReactDOM from 'react-dom';
import App from 'App'

const container = document.getEleementById('root')

const root = ReactDOM.createRoot(container)

root.render(<App />)
