Why Link?

Link from react-router-dom navigates without a full page refresh, preserving SPA performance and state. Use it instead of <a> tags for internal routes.

Routes Setup

Define your app’s route table:

// App.jsx
import { Routes, Route } from "react-router-dom";
import Home from "./components/Home";
import InventoryPage from "./components/InventoryPage";
import Navbar from "./components/Navbar";

export default function App() {
  return (
    <>
      <Navbar />
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/inventory" element={<InventoryPage />} />
      </Routes>
    </>
  );
}

Navigation with Link

Create SPA-friendly links with the to prop:

// components/Navbar.jsx
import { Link } from "react-router-dom";

export default function Navbar() {
  return (
    <nav className="navbar">
      <Link to="/"><button>Home</button></Link>
      <Link to="/inventory"><button>Inventory</button></Link>
    </nav>
  );
}
  • Relationship to Routes: Clicking a Link updates the URL and React Router renders the matching Route’s element—no reload.
  • Prefer Link for internal navigation. Use <a href> only for external sites.

Extras

  • For active styling, use NavLink which adds an active class automatically.
  • For programmatic navigation (e.g., after form submit), use the useNavigate hook.
