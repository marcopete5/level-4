Basic Usage

useNavigate returns a function for programmatic navigation. Invoke it to move to a route after an action (form submit, API success, etc.). 

// components/InventoryItemCard.jsx
import { useNavigate } from "react-router-dom";

export default function InventoryItemCard({ item }) {
  const navigate = useNavigate();

  function openDetails() {
    // Navigate to a specific route
    navigate(`/inventory/${item.id}`);
  }

  return (
    <article className="card">
      <h3>{item.name}</h3>
      <button onClick={openDetails}>View Details</button>
    </article>
  );
}

Navigate Backward / Forward

You can move through browser history by passing a relative delta. 

// components/HistoryControls.jsx
import { useNavigate } from "react-router-dom";

export default function HistoryControls() {
  const navigate = useNavigate();
  return (
    <div className="controls">
      <button onClick={() => navigate(-1)}>Go Back</button>
      <button onClick={() => navigate(1)}>Go Forward</button>
    </div>
  );
}

Options: replace and state

Replace the current history entry or pass transient state to the next route. 

// components/LoginForm.jsx
import { useNavigate } from "react-router-dom";

export default function LoginForm() {
  const navigate = useNavigate();

  async function handleSubmit(e) {
    e.preventDefault();
    // ... authenticate user
    navigate("/dashboard", {
      replace: true,               // prevent back to login
      state: { from: "login" }     // read via useLocation().state
    });
  }

  return (
    <form onSubmit={handleSubmit}>
      <input name="email" />
      <input name="password" type="password" />
      <button>Sign In</button>
    </form>
  );
}

Tips

  • Prefer Link/NavLink for static, visible navigation.
  • Use useNavigate for imperative flows (wizards, success redirects).
  • When redirecting after a mutation, consider replace: true to keep history clean.
